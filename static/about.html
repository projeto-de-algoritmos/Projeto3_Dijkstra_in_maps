This page is a little about me and what I've done.

I like cool algorithms, data structures, and their implementations, and the way they come together to form systems.

Software Engineer at ShortCircuit Inc. (YC W2013)

September 2012 - March 2013

At ShortCircuit, I worked on the real-time mass transit directions app Rover. I was primarily responsible for the backend, which was mainly Python with Flask. Here are a few of the things I created for Rover:

    Real-time mass transit routing algorithm

    The algorithm was implemented based on a recently published research paper. It was a replacement for our older routing algorithm, with a performance increase of up to two orders of magnitude and significant savings in memory. This prevented a backend rewrite from Python to a performance language.

    Walking directions for Open Street Maps

    Shortest path algorithm which provided a visual guide for our users and distance calculations for the transit routing algorithm. Due to the size of OSM data and constraints of Heroku instances, I created a fast, dynamic loading scheme that prevented having to keep the entire graph in memory at once. The algorithm is also capable of handling problems caused by errors in the OSM data set itself, such as unintentional islands in the graph.

    Backend Infrastructure

    I implemented a majority of the current production backend from the ground up. This includes the aforementioned algorithms, logic for handling real-time data from various transit agencies, and tools for processing static data. I was also responsible for the production releases of my code to EC2 and Heroku, including the testing required to make launches go smoothly.

Projects/Awards

Some side projects I've done

    Graph Search Animator

    Animates three graph search algorithms: A* search, Dijkstra's Algorithm, and the more recent A* Landmark Triangle Inequality algorithm, as well as their bidirectional variants. The animator illustrates the vertex scanning behavior that these algorithms take as they explore the graph. Take a look at the link above!

    Best Collaboration at Reroute/sf Hackathon

    Created a heat map showing how far one could travel in 30 minutes in any direction by taking public transit, based on live data of bus positions. I adapted a breadth-first graph search algorithm to do the flood that created the data for the visualization.

    DJ Display (Python/wxPython)

    I created a program for Cheryl Burke Dance which allowed a DJ to display a list of ballroom dance types corresponding with the playing song. The display was timer-based, allowing the DJ to automate the display to match their play list. This replaced an older program which only allowed manual manipulation of the list.
